## 2. 사이클론 테스트 스파이럴

### 🌀 Spiral 0: 스모크 테스트 (기본 접근성)
**목표**: 앱의 기본 기능이 작동하는지 확인

| ID | 테스트 케이스 | 예상 결과 | 상태 |
|----|--------------|----------|------|
| S0-01 | 랜딩 페이지 접근 | 페이지 정상 로드, 로그인 버튼 표시 | ⏳ |
| S0-02 | 이메일 회원가입 | 새 계정 생성 성공, 홈으로 리다이렉트 | ⏳ |
| S0-03 | 이메일 로그인 | 로그인 성공, 사용자 정보 표시 | ⏳ |
| S0-04 | 로그아웃 | 세션 종료, 랜딩 페이지로 이동 | ⏳ |
| S0-05 | 세션 유지 확인 | 새로고침 후에도 로그인 상태 유지 | ⏳ |

### 🌀 Spiral 1: 핵심 학습 루프
**목표**: 사용자 여정 전체 흐름 검증 (대상선택 → 시나리오 → 캐릭터 → 학습)

| ID | 테스트 케이스 | 페르소나 | 예상 결과 | 상태 |
|----|--------------|---------|----------|------|
| S1-01 | 학생 대상 선택 | 리나 | 학생 시나리오 프리셋 표시 | ⏳ |
| S1-02 | 일반인 대상 선택 | 민수 | 일반인 시나리오 프리셋 표시 | ⏳ |
| S1-03 | 비즈니스 대상 선택 | 유지 | 비즈니스 시나리오 프리셋 표시 | ⏳ |
| S1-04 | 프리셋 시나리오 선택 | 리나 | 선택된 시나리오 저장, 다음 단계 이동 | ⏳ |
| S1-05 | 커스텀 시나리오 입력 | 유지 | 자유 텍스트 저장, 다음 단계 이동 | ⏳ |
| S1-06 | 캐릭터 기본 설정 | 민수 | 이름/성별/스타일 설정 가능 | ⏳ |
| S1-07 | 학습 페이지 진입 | ALL | 캐릭터 정보와 시나리오 표시 | ⏳ |

### 🌀 Spiral 2: AI 대화 및 음성 통합
**목표**: AI 대화 생성 및 TTS 음성 기능 검증

| ID | 테스트 케이스 | 페르소나 | 예상 결과 | 상태 |
|----|--------------|---------|----------|------|
| S2-01 | AI 대화 생성 요청 | 리나 | 시나리오에 맞는 일본어 대화 생성 | ⏳ |
| S2-02 | TTS 음성 재생 | 민수 | 캐릭터 목소리로 음성 재생 | ⏳ |
| S2-03 | 시나리오별 맞춤 대화 | 유지 | 비즈니스 맥락에 맞는 대화 시작 | ⏳ |
| S2-04 | 캐릭터 스타일별 목소리 | ALL | cheerful/calm/strict 각각 다른 목소리 | ⏳ |

### 🌀 Spiral 3: 구독 및 사용량 관리
**목표**: 구독 시스템 및 사용량 제한 검증

| ID | 테스트 케이스 | 페르소나 | 예상 결과 | 상태 |
|----|--------------|---------|----------|------|

| S3-02 | 구독 페이지 접근 | 소라 | 요금제 목록 표시 | ⏳ |
| S3-03 | 사용량 카운트 증가 | 민수 | 학습 후 사용량 +1 | ⏳ |

| ID | 테스트 케이스 | 페르소나 | 예상 결과 | 상태 |
|----|--------------|---------|----------|------|

## 3. 동적 데이터 전략 (하드코딩 방지)

### 테스트 데이터 생성 규칙
```javascript
// 고유한 테스트 데이터 생성
const testUser = {
  email: `test_${Date.now()}@example.com`,
  password: `TestPass_${Math.random().toString(36).slice(2)}`,
  firstName: `User_${nanoid(6)}`,
  lastName: `Test_${nanoid(6)}`
};
```

### API 기반 시드 데이터
- 사용자 생성: POST /api/register
- 캐릭터 저장: POST /api/saved-characters
- 시나리오 선택: 앱 스토어 상태 활용

- 테스트 사용자 삭제 (관리자 API 또는 직접 DB)
